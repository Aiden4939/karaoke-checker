# Review Change

1. Read the issue, spec, and diff.
2. Confirm the change matches the requested scope.
3. Check for accidental business-logic or UI changes.
4. Review error handling, validation, and typing.
5. Look for missing tests or verification gaps.
6. Classify findings as Critical, Major, or Minor.
7. Provide file-specific remediation guidance.
8. Do not stop at "looks fine"; state what was checked.
