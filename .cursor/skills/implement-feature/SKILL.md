# Implement Feature

1. Read the issue, spec, and relevant rules in `docs/` and `.cursor/rules/`.
2. Search for similar existing code.
3. Confirm requirement boundaries and out-of-scope items.
4. List files you expect to change.
5. Implement the smallest complete change.
6. Add or update unit, integration, or E2E tests as needed.
7. Run `pnpm verify` or `pnpm verify:full`.
8. For UI work, validate in a browser and check console/network.
9. Summarize changes, test results, and residual risks.
